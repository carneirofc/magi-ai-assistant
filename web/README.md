# MAGI Admin (web)

Operator dashboard + BFF for MAGI memory & knowledge management. Next.js (App
Router), the browser-facing half of the admin service described in
[`docs/adr/0002`](../docs/adr/0002-admin-interface-for-memory-and-knowledge.md).

It is a **BFF**: the browser only ever talks to this Next.js server, which holds
the admin-api bearer token server-side and gates the operator with a password →
httpOnly session cookie. The token never reaches the browser, and the Python
`admin-api` is never exposed publicly (compose-network only).

## Design system

The UI is built on [`@carneirofc/ui`](https://github.com/carneirofc/deedlit.dev/tree/master/deedlit.dev.ui)
— a Tailwind v4 + Radix component library (the "cyber" neon token system, with
light/dark themes). Tailwind runs as a PostCSS plugin
([`postcss.config.mjs`](postcss.config.mjs)); the tokens, fonts, and `.cyber-*`
component classes come from `@carneirofc/ui/styles.css`, imported once in
[`src/app/globals.css`](src/app/globals.css) alongside an `@source` pointing at
the kit's compiled output so Tailwind keeps the utility classes those components
emit. Theme is `data-theme` on `<html>`, initialised flash-free in the root
layout and toggled by the kit's draggable `ThemeToggleButton`.

### Installing `@carneirofc/ui` (GitHub Packages)

The package is published to **GitHub Packages**, which requires auth even for
public packages. [`.npmrc`](.npmrc) scopes `@carneirofc/*` to that registry and
reads the token from `NODE_AUTH_TOKEN`. Create a GitHub **Personal Access Token
(classic)** with the `read:packages` scope (github.com → Settings → Developer
settings → Tokens — the `gh` CLI token does *not* carry this scope), then:

```bash
export NODE_AUTH_TOKEN=ghp_xxx      # PowerShell: $env:NODE_AUTH_TOKEN = "ghp_xxx"
npm install
```

## Develop

```bash
cp .env.example .env.local        # set ADMIN_API_URL, ADMIN_AUTH_TOKEN, ADMIN_PASSWORD, SESSION_SECRET
# start the Python admin-api in another shell: `python main_admin.py` (binds :8100)
export NODE_AUTH_TOKEN=ghp_xxx     # read:packages PAT (see above)
npm install
npm run gen:api                   # generate src/lib/api-types.ts from the live admin-api OpenAPI
npm run dev                       # http://localhost:3000
```

## Configuration (server-side only — never shipped to the browser)

| Var | Purpose |
| --- | --- |
| `ADMIN_API_URL` | Base URL of the Python admin-api (e.g. `http://admin-api:8100` in compose). |
| `ADMIN_AUTH_TOKEN` | Bearer the BFF presents to admin-api. Must equal admin-api's `ADMIN_AUTH_TOKEN`. |
| `ADMIN_PASSWORD` | The single operator password the login screen checks. |
| `SESSION_SECRET` | HMAC secret for signing the session cookie. |
| `NODE_AUTH_TOKEN` | **Build-time only.** GitHub PAT (`read:packages`) to install `@carneirofc/ui`. Not read at runtime. |

## Layout

```
src/
  middleware.ts                       # auth gate → /login unless a valid session cookie
  lib/
    session.ts                        # signed httpOnly cookie (Web Crypto HMAC)
    admin-api.ts                      # server-only client; holds the bearer
    api-types.ts                      # GENERATED by `pnpm gen:api` (placeholder committed)
  components/
    AppShell.tsx / Sidebar.tsx        # dashboard chrome: fixed left rail (active nav) + content
    StatCard.tsx                      # dashboard metric tile
    FactEditor / RawFileEditor        # memory editors (facts as cards, raw files)
    KnowledgeList / AddKnowledge      # corpus table+cards / ingest form
    DocumentActions / DocumentMeta    # rename+delete / subject+tags
    SubjectManager                    # controlled-vocabulary CRUD
  app/
    layout.tsx                        # root: theme init + global ThemeToggleButton
    globals.css                       # Tailwind v4 + @carneirofc/ui styles + app-shell CSS
    login/page.tsx                    # password login (chrome-free, outside the (app) group)
    (app)/                            # authed dashboard group — wrapped in AppShell
      layout.tsx                      # sidebar shell
      page.tsx                        # Dashboard: stat tiles + recent docs + top users
      knowledge/page.tsx              # document list (table/cards) + subject/tag filters
      knowledge/add/page.tsx          # add by paste / upload
      knowledge/[...docId]/page.tsx   # document detail: rename/delete, subject/tags, chunks
      subjects/page.tsx               # manage the controlled-subject vocabulary
      memory/page.tsx                 # user cards
      memory/[user]/page.tsx          # tabs: facts (cards, CRUD) / episodes / sessions
      memory/[user]/sessions/[sid]/page.tsx  # edit window/summary/pending (raw)
      persona/page.tsx                # edit the global persona (raw)
    api/auth/{login,logout}/route.ts  # session set/clear
    api/admin/**/route.ts             # BFF mutation proxies (docs, subjects, facts, raw, ingest)
```

## Docker

Built as a standalone Node server (`output: "standalone"`). The `@carneirofc/ui`
install needs the `read:packages` token, passed as a **build secret** so it never
lands in an image layer:

```bash
export NODE_AUTH_TOKEN=ghp_xxx
docker build --secret id=node_auth_token,env=NODE_AUTH_TOKEN -t magi-admin-web:local ./web
```

Or with the app stack under the `admin` profile (wire the same secret into your
compose build):

```bash
docker compose -f docker-compose.app.yaml --profile admin up --build
# web → http://localhost:3000 ; admin-api stays internal to the compose network
```
